Faille N°1 : Gain d'accès (Déobfuscation)

Pour cette première vulnérabilité c'est au niveau du javascript que ça va se jouer. Commence par ouvrir les outils développeurs de ton naviguateur (Touche F12). Puis va dans l'onglet sources et dans le dossier js. Clique sur le fichier script.js.


On peut voir la première variable "op" qui stocke une chaine de caractères obfusqué. Et en dessous la fonction "dpf(obfuscated)" qui permet une désobfuscation. Désormais nous avons compris la logique du script, il n'y a pas qu'à l'exploiter !


Faille N°2 : Cracker le hash (Escalade de privilèges)

Pour la seconde vulnérabilité il s'agit d'un hash, soit un mot de passe crypté via un algorithme, pour savoir de quel type de hash il s'agit on se rend sur hashes.com et le résultat de l'identification est MD5.

Maintenant il faut cracker le hash pour obtenir le mot de passe en clair, on va se rendre sur crackstation.net ou utiliser un crackeur de hash md5 sur github pour obtenir le mot de passe.


Faille N°3 : Osint (Accès privilégié direct)

Pour cette troisième faille je vous laisse la trouver par vous même. Ce n'est pas une vulnérabilité qui requiert des connaissances techniques mais juste de la logique. L'osint est la recherche d'informations et l'exploitation de vulnérabilités humaine. Tel qu'un nom, un email, un numéro ou encore un mot de passe lié à sa vie privée...
